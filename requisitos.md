## 3. Requisitos Funcionais (RF)

Definem o que o sistema deve fazer.

- RF01: O sistema deve permitir o cadastro e login de usuários, diferenciando leitores e críticos.
- RF02: O sistema deve permitir a publicação de reviews estruturadas, contendo notas por categoria (ex: enredo, personagens, animação, trilha sonora).
- RF03: O sistema deve oferecer um mecanismo de busca que permita filtrar animes por título, gênero, estúdio ou ano de lançamento.
- RF04: O sistema deve permitir que usuários criem e gerenciem watchlists (listas de interesse).
- RF05: O sistema deve calcular e exibir uma média ponderada entre a avaliação técnica dos críticos e a opinião do público.

## 4. Requisitos Não Funcionais (RNF)

Definem as qualidades, restrições e padrões do sistema.

- RNF01 (Performance): O carregamento das páginas de review não deve ultrapassar 2 segundos, visando a retenção do usuário.
- RNF02 (Responsividade): A plataforma deve ser totalmente responsiva e otimizada para dispositivos móveis.
- RNF03 (Escalabilidade): O banco de dados deve suportar o crescimento contínuo do catálogo de animes e picos de acesso em períodos de lançamento.
- RNF04 (SEO): A estrutura do sistema deve ser otimizada para mecanismos de busca, garantindo melhor posicionamento das reviews nos resultados do Google.
